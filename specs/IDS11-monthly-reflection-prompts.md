# SPEC: Monthly reflection prompts

**Depends on:** IDS08 (period performance and benchmark outcomes), IDS09 (closed-trade statistics and rule-adherence retrospective). Optional dependency on IDS03 (position log) for prompts about rationale hygiene; this SPEC tolerates a missing position log and adapts prompts accordingly.

## User Story

As an **active investor running the build–implement–evaluate–modify cycle**, I want **the monthly report to end with a small set of structured reflection prompts populated from this period's data**, so that **I have a recurring, pointed nudge to think about what to change in the strategy instead of letting evaluation data flow past me**.

## Context

The strategy file is read-only for the system (per PRD section 4). The user is the sole author of strategy modifications. But authorship is only useful if it is exercised — and the user's stated problem is precisely that the evaluate-modify part of the cycle does not happen, because nothing forces it. This SPEC provides that forcing function in the lightest possible form: a Markdown section at the bottom of the monthly report containing 3–6 questions tied to concrete observations from the period.

The prompts are **deterministic and template-driven**, not generated by an LLM. Two reasons:

1. **Authorship boundary.** A free-form LLM prompt risks proposing strategy changes (which is out of scope per PRD product borders). Templates with placeholders only restate the data and ask a question — they cannot accidentally suggest a modification.
2. **Predictability and reviewability.** The user can anticipate what categories of prompts will appear, which lowers the cognitive cost of reading the report. Surprises in a self-directed learning loop are usually friction, not value.

Each template is associated with a **trigger condition**. A prompt is included only if the period's data satisfies its trigger (e.g., "rule adherence" prompt fires only when at least one closed trade breached −5%). This avoids vacuous prompts when nothing meaningful happened in a category.

## Acceptance Criteria

### Section presence and placement

- [ ] GIVEN the monthly report has been generated through IDS10
      WHEN the reflection prompts section is added
      THEN the report contains a `## Reflection prompts` section placed at the end of the report (after the charts section)
      AND existing sections from IDS08, IDS09, and IDS10 remain unchanged in structure and content

- [ ] GIVEN no template's trigger condition fires for the period
      WHEN the report is generated
      THEN the `## Reflection prompts` section is still present
      AND it explicitly states that this period produced no prompts (e.g., "No reflection prompts this period — every category's trigger condition was met cleanly")

### Rule-adherence prompt

- [ ] GIVEN at least one closed trade in the period exceeded the −5% threshold before being closed (data from IDS09)
      WHEN the reflection section is generated
      THEN a rule-adherence prompt appears
      AND it names the specific symbols and references their close-loss percentages
      AND it asks whether the deviations were deliberate and whether the strategy needs to encode an exception

- [ ] GIVEN no closed trade in the period exceeded the −5% threshold
      WHEN the reflection section is generated
      THEN the rule-adherence prompt is omitted

### Outperformance prompt

- [ ] GIVEN the portfolio's period return is ahead of at least one benchmark (data from IDS08)
      WHEN the reflection section is generated
      THEN an outperformance prompt appears
      AND it states the benchmark and the lead in percentage points
      AND it asks which positions drove the lead and whether the signals are repeatable

- [ ] GIVEN the portfolio's period return is at or below every benchmark
      WHEN the reflection section is generated
      THEN the outperformance prompt is omitted

### Underperformance prompt

- [ ] GIVEN the portfolio's cumulative return is behind at least one benchmark (data from IDS08)
      WHEN the reflection section is generated
      THEN an underperformance prompt appears
      AND it states the benchmark and the cumulative gap in percentage points
      AND it asks what would need to change for active management to be worth the time spent versus leaving capital in passive instruments

- [ ] GIVEN the portfolio's cumulative return matches or exceeds every benchmark
      WHEN the reflection section is generated
      THEN the underperformance prompt is omitted

### Position-log hygiene prompt

- [ ] GIVEN the position log directory exists and at least one closed position in the period has an empty `## Close rationale` section (data shape from IDS03 / IDS04)
      WHEN the reflection section is generated
      THEN a hygiene prompt appears stating how many closed positions in the period are missing rationale
      AND it asks whether the rationale capture cadence is working

- [ ] GIVEN the position log directory does not exist (e.g., IDS03 has not been delivered yet)
      WHEN the reflection section is generated
      THEN the hygiene prompt is omitted
      AND no error is raised

- [ ] GIVEN every closed position in the period has its close rationale filled
      WHEN the reflection section is generated
      THEN the hygiene prompt is omitted

### Prompt quantity

- [ ] GIVEN multiple template triggers fire
      WHEN the reflection section is generated
      THEN no more than six prompts are produced
      AND prompts are ordered with rule-adherence first, performance prompts (out / under) next, hygiene last

### Determinism

- [ ] GIVEN the same input data
      WHEN the reflection section is regenerated
      THEN the resulting prompts are identical (same wording, same order, same placeholders) — no randomness or LLM-driven variability

## Out of Scope

- LLM-generated or paraphrased prompts — explicitly out per the authorship boundary; templates are static
- Suggested strategy modifications, recommended rule changes, or any prescriptive language — the system asks questions only, never proposes answers
- Prompts based on data the system does not yet collect (e.g., emotional state, market context, news flow) — out of MVP
- A history of past prompts and whether the user acted on them — out of MVP; reflection acts are private
- Prompt categories beyond the four listed (rule adherence, outperformance, underperformance, hygiene) — additional categories can be added in later SPECs
- Customisation of prompt templates by the user — templates are system constants in MVP
- Quarterly strategy-modification report — explicitly out of MVP per PRD product borders
